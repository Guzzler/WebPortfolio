# Portfolio Daily Plan - 2026-03-13

## Chosen Improvement
Design and implement a more personal "control room" introduction system that replaces the current generic sidebar-first impression with an intentional landing area combining Sharang's technical identity, current role, social-impact motivation, and playful interests.

## Why This Matches Sharang's Identity
- The current site communicates strong credentials, but the visual framing is close to a polished SaaS template rather than a serious engineer with a creative, game-building side.
- Sharang's story is strongest when AI/product depth, entrepreneurship, human-computer interaction, low-resource design, and social relevance are presented together instead of split across isolated sections.
- A control-room style intro can credibly connect Luma AI product engineering, prior founding-ML work, CMU AI training, and game/sports/board-game interests without inventing new facts.
- This improvement strengthens authenticity and distinctiveness while preserving the existing one-page portfolio and dark-mode-capable experience.

## Implementation Notes
- Reframe the top of `BaseContent` so the first screen reads as an introduction canvas, not just a sidebar plus resume columns.
- Convert the sidebar content into a stronger identity block: name, current role at Luma AI, concise mission statement, and 2-4 factual personal signal chips such as AI systems, social-impact technology, games, sports, or board games.
- Add a short "currently building / cares about / plays with" cluster that creates fast narrative structure without bloating copy.
- Use the existing portrait and dark mode, but move away from the current blue-slate enterprise aesthetic toward a more intentional visual language inspired by notebooks, dashboards, or game HUD elements.
- Keep the experience timeline and game gallery in the same page, but visually connect them to the intro so games feel like part of the identity rather than an isolated add-on.
- Improve mobile hierarchy so the intro stack is compact, readable, and not dominated by oversized empty space or secondary UI chrome.
- Tighten accessibility at the same time: clearer heading hierarchy, stronger button/link focus states, and text/background contrast that remains valid in both modes.

## Acceptance Criteria
- The first viewport clearly communicates who Sharang is, what he is building now, and why his work matters before the user scrolls into resume details.
- The intro feels personal and specific to Sharang rather than interchangeable with a startup marketing page.
- Games, sports, and board-game interests are integrated as authentic identity signals, not novelty decoration.
- Desktop layout feels more intentional and visually distinctive while preserving readability and scannability.
- Mobile layout preserves the same narrative without overflow, cramped columns, or awkward spacing.
- Dark mode and light mode both remain usable with clear contrast and visible interaction states.
- No factual claims are added beyond what is already supported by current content and the owner profile supplied to the automation.

### Mobile

2. Confirm the intro stack reads cleanly without horizontal scroll or overly tall dead space.
3. Verify that the timeline, education cards, skills, hobbies, and game gallery still stack naturally after the intro.
4. Toggle dark mode and confirm text contrast, buttons, and icon controls remain clear.
5. Check that the game gallery still works as expected on mobile with swipe behavior and external-link handling.

## Risks To Avoid
- Content inaccuracy about Sharang's roles, dates, motivations, or technical focus.
- Overdesigned visuals that distract from credibility or make the site feel like a game instead of a professional portfolio.
- Broken responsiveness caused by forcing a desktop-first hero into the existing one-page layout.
- Generic AI-generated styling patterns, especially abstract blue corporate gradients with little relation to Sharang's actual story.
- Letting the intro crowd out the experience timeline or bury the game gallery behind unnecessary chrome.
